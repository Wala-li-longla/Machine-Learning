hexo+github+zeabur 搭建个人博客网站

Hexo官网

  • Hexo 是一个博客框架。
  • Hexo 使用 Markdown（或其他标记语言）解析文章，生成静态网页，并自带靓丽的主题。

Zeabur官网

博客网站教程

Hexo用法

1. 打开命令提示符（win+R-输入cmd）

2. 使用 npm / cnpm 安装 Hexo

3. 使用 hexo 命令 - hexo init 初始化目录

（1）创建文件夹 hexo

（2）进入 hexo 文件夹，地址栏输入 cmd，然后输入 hexo init

（3）或者可以在一些编程软件中（如 VS code）新建终端，然后输入 hexo init

执行一段时间后，会卡住不动，这是因为初始化目录的安装包需要连接外网，所以直接 Ctrl+c 退出，再继续执行 cnpm install 即可

4. 输入命令 hexo g

生成静态文件，会在当前目录下生成一个新的叫做public的文件夹

5. 输入命令 hexo s

启动本地web服务，用于博客的本地预览

  • 执行该命令后，会生成一个地址，复制到浏览器打开，会看到hexo文件夹初始化后的界面

  • 这只是一个基础框架，还未使用任何主题

发布到Github

1. 创建github仓库Hexo-Blog

2. config.yaml配置github仓库

![[Pasted image 20240813012151.png]]

3. 上传到github仓库

（1）先执行npm install --save hexo-deployer-git
（2）再依次执行hexo cl、hexo g、hexo d
（3）hexo d 可能会失败，可以执行git config –global http.sslVerify false，再执行hexo d
（4）结束后记得执行git config –global http.sslVerify true

hexo文件夹发布到github上的目录是不一样的

![[Pasted image 20240813014343.png]]

使用 Zeabur 部署到互联网

1. 进入zeabur官网，得到github授权

2. 新建项目，配置github仓库hexo-blog

3. 点击网络，自动生成一个免费的域名（一个仓库对应一个域名）

![[Pasted image 20240813013132.png]]

4. 访问该域名，即可访问本地hexo文件夹里的内容

因为 zeabur 得到了 github 的授权，所以只要执行 hexo d 更新 github 库，访问 zeabur 生成的域名，显示的内容就会更新

==本地图片如何处理==
除了背景图片由固定的存放路径以外，其他的图片都需要放在 public/img 里，这样才会发布到 github 上。实际上背景图片在上传github时也会复制一份存放在这个文件夹里。
